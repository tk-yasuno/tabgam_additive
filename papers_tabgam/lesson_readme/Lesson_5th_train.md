## 6. 🎓 重要な教訓

### 教訓1: アンサンブルの「多様性」の意味

**誤解**:
- 多くのモデルを組み合わせれば良い
- 全モデルに均等に重み付け

**正解**:
- **相補的な2-3モデル**が最適
- GAM（統計）+ TabNAM（深層学習）の組み合わせ
- FAM、GBAMは冗長（TabNAMでカバー）

**教訓**: **Diversityは数ではなく質**

### 教訓2: 最適化手法の収束速度

**観察**:
- イテレーション1でほぼ最適解
- Differential Evolution でも同じ解
- ランダム摂動でも改善なし

**理由**:
- 重み最適化は凸問題に近い
- 局所最適が大域最適

**教訓**: **単純な問題には単純な手法**

### 教訓3: モデル個別性能とアンサンブル貢献

| モデル | 個別R² | 重み | 貢献度 |
|--------|--------|------|--------|
| TabNAM | 0.2150（1位） | 80% | 高 |
| GBAM | 0.2045（2位） | 0% | なし |
| GAM | 0.2043（3位） | 20% | 中 |
| FAM | 0.1880（4位） | 0% | なし |

**教訓**: **個別性能≠アンサンブル貢献**
- GBAMは2位だが不採用
- GAMは3位だが採用（TabNAMとの補完性）

### 教訓4: Agentic AI の適用範囲

**適している問題**:
- ✅ 探索空間が広い（多数のモデル、HP）
- ✅ 評価に時間がかかる（CV、大規模データ）
- ✅ 人間の直感が効きにくい

**適していない問題**:
- ❌ 探索空間が狭い（2-3モデルのみ）
- ❌ 評価が瞬時（単純な関数）
- ❌ 最適解が自明

**v0.5の場合**:
- ⚠️ **4モデルのみ** → 探索空間が比較的狭い
- ✅ **自動化のメリット** → 再現性、拡張性

**教訓**: **v0.5はAgentic AIの練習台、v0.6でHP最適化に拡張**

## 9. 📚 技術的詳細

### AgenticTuner の実装

**主要クラス**:
```python
class AgenticTuner:
    def plan_next_configuration(self, iteration)
    def execute_configuration(self, config)
    def evaluate_iteration(self, iteration, weights, score)
    def run(self) -> results
```

**探索手法**:
1. **Uniform**: 均等重み（ベースライン）
2. **SLSQP**: 制約付き最適化（局所）
3. **Differential Evolution**: 遺伝的アルゴリズム（大域）
4. **Random Perturbation**: ベスト重みからの摂動

**終了条件**:
- Max iterations: 50
- Patience: 10（改善なしで打ち切り）

### 計算複雑度

- **モデル訓練**: O(n × m) - n:サンプル数、m:特徴量数
- **重み最適化**: O(k × e) - k:イテレーション数、e:アンサンブルサイズ
- **総計**: 約30-60秒（45,058サンプル、73特徴量、4モデル）
